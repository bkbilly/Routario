Routario REST API

API Keys

Use API keys as Bearer tokens. Create keys from User Settings → API Keys. Keys are scoped, can expire, and are shown only once.

Authorization: Bearer rt_your_api_key_here
Content-Type: application/json

Scopes

ScopePurpose
devices:readRead devices and state.
devices:writeCreate or update devices.
positions:readRead position history.
commands:sendSend commands to supported devices.
reports:readRun and read reports.
routes:readRead planned routes.
routes:writeCreate, update, and delete planned routes.
billing:readRead billing usage and invoices.

Common Endpoints

MethodPathDescription
GET/api/devicesList devices visible to the key owner.
GET/api/devices/{device_id}/stateGet current device state.
POST/api/positions/historyRead historical positions for a device/date range.
GET/api/reports/typesList available backend-defined reports.
GET/api/planned-routesList planned routes.
POST/api/planned-routes/previewPreview route geometry from ordered stops.
POST/api/planned-routesCreate a route with ordered stops.
PUT/api/planned-routes/{route_id}Update route assignment, status, or stops.
DELETE/api/planned-routes/{route_id}Delete a planned route.
GET/health/readyReadiness check for monitoring.

Route Example

curl -X POST https://your-host/api/planned-routes \
  -H "Authorization: Bearer rt_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Morning route",
    "device_id": 12,
    "stops": [
      {"sequence": 0, "name": "Depot", "latitude": 37.9838, "longitude": 23.7275},
      {"sequence": 1, "name": "Customer", "latitude": 38.0500, "longitude": 23.8000}
    ]
  }'

Routes with an assigned device_id are saved as planned. Unassigned routes are saved as draft. Route geometry uses Valhalla when available and falls back to straight-line geometry.
